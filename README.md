# Лабораторная работа №22: Интерактивные веб-приложения на JavaScript

## Описание проекта

Данный проект представляет собой два интерактивных веб-приложения, разработанных в рамках лабораторной работы №22:

1. **Калькулятор** — выполняет базовые арифметические операции с валидацией ввода и обработкой ошибок
2. **Крестики-нолики** — классическая игра для двух игроков с определением победителя и ничьей

Проект демонстрирует использование современных технологий веб-разработки: HTML5, CSS3 (Flexbox, Grid), JavaScript (ES6+), а также модульную архитектуру с применением ES6 Modules.

---

## Функциональность

### Калькулятор

#### Поддерживаемые операции

| Операция | Символ | Пример |
|----------|--------|--------|
| Сложение | `+` | `5 + 3 = 8` |
| Вычитание | `-` | `10 - 4 = 6` |
| Умножение | `*` | `7 * 6 = 42` |
| Деление | `/` | `15 / 3 = 5` |

#### Ограничения ввода

- Нельзя вводить два оператора подряд (`5++3`)
- Нельзя начинать выражение с оператора (`+5`)
- Нельзя вводить несколько точек в одном числе (`5.2.3`)
- Выражение не может заканчиваться оператором (`5+3-`)

#### Обработка ошибок

| Ситуация | Результат |
|----------|-----------|
| Деление на ноль | Показывает `Ошибка` |
| Некорректное выражение | Показывает `Ошибка` |
| Состояние ошибки | Сбрасывается при нажатии любой кнопки |

### Крестики-нолики (Tic-Tac-Toe)

#### Игровая механика

- Игровое поле размером **3×3**
- Два игрока: **X** и **O**
- Поочерёдные ходы
- Автоматическое определение победителя
- Определение ничьей при заполнении всех ячеек
- Кнопка **"Новая игра"** для сброса состояния

#### Условия победы

| Тип | Комбинации |
|-----|------------|
| Строки | (0,1,2), (3,4,5), (6,7,8) |
| Столбцы | (0,3,6), (1,4,7), (2,5,8) |
| Диагонали | (0,4,8), (2,4,6) |

## Логика работы

### Последовательность действий при нажатии кнопок

1. **Клик по кнопке** → вызов `handleButtonClick(value)`
2. **Проверка состояния ошибки** → если ошибка и кнопка не "C", выполняется очистка
3. **Определение типа кнопки**:
   - "C" → вызов `clear()`
   - "=" → вызов `calculate()`
   - Другое → вызов `appendValue(value)`

### Валидация ввода

```javascript
// 1. Запрет оператора в начале
if (this.expression === '' && this.isOperator(value)) return;

// 2. Запрет двух операторов подряд
if (this.isOperator(value) && this.isLastCharOperator()) return;

// 3. Запрет нескольких точек в числе
if (value === '.' && this.hasDuplicateDot()) return;
```
### Вычисление

// 1. Проверка корректности выражения
isValidExpression(expr) → проверка:
  - отсутствие двойных операторов
  - выражение не заканчивается оператором
  - корректное использование точек

// 2. Безопасное вычисление
safeEvaluate(expr) → использует Function() вместо eval
  - обрабатывает ошибки
  - проверяет NaN и Infinity
  - возвращает null при ошибке


# Лабораторная работа №23. Интерактивные веб-приложения на JavaScript

## Описание проекта

Данный проект представляет собой два интерактивных веб-приложения, разработанных в рамках лабораторной работы №23:

1. **Крестики-нолики (Tic-Tac-Toe)** — классическая игра для двух игроков с определением победителя и ничьей
2. **Профиль пользователя** — интерактивная страница с формой для ввода и отображения пользовательских данных

Проект демонстрирует использование современных технологий веб-разработки: HTML5, CSS3 (Flexbox, Grid), JavaScript (ES6+), а также работу с DOM-элементами, событиями и валидацией данных.

---

## Игра: Крестики-нолики

### Правила игры

1. Игра ведётся на поле размером **3×3**
2. Первый ход делает игрок **X**
3. Игроки поочерёдно ставят свой символ (X или O) в пустые клетки
4. Побеждает игрок, первым выстроивший **3 своих символа** в ряд:
   - По горизонтали
   - По вертикали
   - По диагонали
5. Если все 9 клеток заполнены, но победитель не определён, объявляется **ничья**

### Логика победы и поражения

- Победа определяется при совпадении символов в одной из 8 выигрышных комбинаций
- После победы игра блокируется, дальнейшие клики на клетки не обрабатываются
- Ничья наступает, когда все клетки заполнены и нет победителя

### Особенности реализации

- **Визуальная обратная связь** — при наведении на клетку меняется цвет фона
- **Блокировка игры** после победы или ничьей
- **Кнопка перезапуска** для начала новой партии без перезагрузки страницы
- **Отображение статуса** — показывается текущий игрок или результат игры
